📖 Documentation de l'API
1

Générez votre clé API

Cliquez sur "Générer une clé" dans la section ci-dessus. Copiez-la immédiatement — elle ne sera plus visible en clair ensuite. Donnez-la à votre technicien caisse uniquement.

2

Configurez votre caisse

Dans votre logiciel, cherchez "Webhook", "Intégration HTTP" ou "Export automatique". Renseignez :

  • URL : https://alteore.com/api/public-ca
  • Méthode : POST
  • Header : Authorization: Bearer {votre_clé}
  • Content-Type : application/json
  • Déclencheur : Z de caisse (clôture journalière)
3

Vérifiez dans le Pilotage

Après la première clôture, ouvrez Pilotage → Saisie du CA. Vos données doivent apparaître automatiquement pour la journée concernée.

# Test depuis un terminal — remplacez alte_votreclé... par votre vraie clé curl -X POST https://alteore.com/api/public-ca \ -H "Authorization: Bearer alte_votreclé..." \ -H "Content-Type: application/json" \ -d '{ "date": "2026-03-04", "ht055": 0, "ht10": 150.00, "ht20": 320.50 }' # Réponse attendue (HTTP 200) { "ok": true, "updated": ["4/3/2026"], "message": "1 jour(s) mis à jour" }
📌 Format 1 — Multi-TVA (recommandé)
Idéal pour les caisses ventilant le CA par taux de TVA (restauration, commerce mixte…)
{ "date": "2026-03-04", // ISO, FR (4/3/2026) ou DD-MM-YYYY "ht055": 0, // CA HT à 5,5% — alimentation, livres, presse "ht10": 150.00, // CA HT à 10% — restauration, transport, hébergement "ht20": 320.50 // CA HT à 20% — taux standard }
📌 Format 2 — Montant unique + taux TVA
Pour les caisses n'appliquant qu'un seul taux de TVA
{ "date": "2026-03-04", "montantHT": 470.50, // Montant HT total (pas TTC !) "tvaRate": 20 // 5.5 | 10 | 20 }
📌 Format 3 — Tableau multi-jours
Pour importer plusieurs jours en un seul appel (max 100 lignes)
{ "lignes": [ { "date": "2026-03-03", "ht10": 120, "ht20": 280 }, { "date": "2026-03-04", "ht10": 150, "ht20": 320.50 } ] } // Maximum 100 lignes par appel
📌 Format 4 — Mode "add" (cumul)
Pour additionner un montant à la valeur déjà existante (plusieurs services dans la journée)
{ "date": "2026-03-04", "ht20": 120.00, "mode": "add" // "replace" (défaut) ou "add" (cumule) }
📐 Comment calculer le HT depuis le TTC ?

Alteore attend toujours des montants HT. Votre caisse peut avoir des totaux TTC. Voici la formule selon le taux :

Taux TVASecteurs concernésFormule TTC → HTChamp Alteore
5,5% Épicerie, boulangerie, livres, presse, abonnements énergie HT = TTC ÷ 1.055 ht055
10% Restauration (repas), transport de personnes, travaux d'entretien HT = TTC ÷ 1.10 ht10
20% Taux standard — la plupart des services, vêtements, électronique… HT = TTC ÷ 1.20 ht20
💡
Bonne nouvelle : la plupart des caisses modernes (Zelty, Lightspeed, L'Addition…) exposent directement le CA HT dans leurs rapports de Z de caisse. Vous n'avez donc généralement pas à faire le calcul vous-même — il suffit de mapper le bon champ.
🍽️ Exemple restauration classique

Un restaurant avec boissons (20%) et repas (10%) :

{ "date": "2026-03-04", "ht10": 1840.00, // Repas = 2024 TTC ÷ 1.10 "ht20": 450.00 // Boissons = 540 TTC ÷ 1.20 }
🏪 Exemple commerce alimentaire mixte

Épicerie avec frais/sec (5,5%) et produits non-alimentaires (20%) :

{ "date": "2026-03-04", "ht055": 620.00, // Alimentaire = 654.10 TTC ÷ 1.055 "ht20": 180.00 // Non-alimentaire = 216 TTC ÷ 1.20 }

❓ Questions fréquentes
Ma caisse envoie des montants TTC — comment faire ?›

Divisez par 1.055 (TVA 5,5%), 1.10 (TVA 10%) ou 1.20 (TVA 20%) avant d'envoyer. La plupart des caisses modernes ont un rapport Z de caisse qui inclut déjà le HT. Consultez votre technicien caisse pour mapper directement le champ HT.

J'ai plusieurs services dans la journée (midi + soir) — dois-je envoyer deux fois ?›

Deux options : (1) envoyez une seule fois en fin de journée avec le total de la journée — c'est la méthode recommandée, (2) envoyez après chaque service en utilisant "mode": "add" pour cumuler les montants sans écraser le premier envoi.

Que se passe-t-il si je renvoie des données pour un jour déjà saisi ?›

Par défaut (mode "replace"), les nouvelles données écrasent les anciennes pour ce jour. C'est utile si vous voulez corriger une erreur. Pour ajouter au lieu de remplacer, utilisez "mode": "add".

Ma caisse n'a pas de webhook — que faire ?›

Utilisez l'import CSV manuel en bas de page. Exportez votre rapport de Z de caisse en CSV depuis votre logiciel, renommez les colonnes (date, ht055, ht10, ht20) et importez directement ici. C'est moins automatique mais tout aussi efficace.

Puis-je tester sans impacter mes vraies données ?›

Oui — envoyez un CA de test sur une date dans le futur (ex: le 31/12/2099). Vos données actuelles ne seront pas touchées. Vous pourrez ensuite supprimer cette entrée manuellement dans le Pilotage.

La clé API est-elle liée à mon abonnement ?›

Oui. Si votre abonnement expire ou passe en past_due, les envois depuis votre caisse seront rejetés avec une erreur PLAN_RESTRICTED. Vos données existantes ne sont pas affectées. Renouvelez votre abonnement pour reprendre les envois.

📂 Import CSV manuel
Pour les caisses sans webhook ou rattrapage de données

Format attendu : une ligne par jour avec les colonnes date, ht055, ht10, ht20. La 1ère ligne doit être l'en-tête.

date,ht055,ht10,ht20 2026-03-01,0,120.50,280.00 2026-03-02,0,95.00,310.50 2026-03-03,50.00,110.00,260.00
